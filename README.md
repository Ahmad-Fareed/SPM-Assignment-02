## 📝 Project Overview
This repository contains the project developed as part of the **Fundamentals of Software Project Management (SPM)** assignment. 

The primary focus of this project is to demonstrate a practical **Agile workflow** by integrating **Jira** with **GitHub**. It showcases end-to-end project management, including issue tracking, sprint planning, and synchronized development execution.

---

## 🎯 Objectives
* **Epic & Backlog Management:** Create, organize, and manage Epics and product backlog items in Jira.
* **Sprint Execution:** Plan, structure, and execute agile sprints effectively.
* **CI/CD & Tool Integration:** Seamlessly integrate Jira Software with GitHub.
* **Traceable Development:** Track development cycles using Git branches, commits, and Pull Requests (PRs) directly linked to Jira issue keys.

## 🔄 Agile Workflow Implemented
This project strictly follows an industry-standard Agile lifecycle:

1. **Planning:** Create Epics and define granular backlog items in Jira.
2. **Sprint Setup:** Plan the active sprint and assign tasks to team members.
3. **Branching:** Create feature branches in Git using the exact Jira issue keys (e.g., `feature/SPM-12-authentication`).
4. **Development & Commits:** Commit code with messages that reference the Jira issues (e.g., `git commit -m "SPM-12: Added user login logic"`).
5. **Review & Merge:** Create Pull Requests on GitHub and merge them into the main branch post-review.
6. **Tracking:** Monitor real-time progress, branch statuses, and merged PRs via the Jira Development Panel.
